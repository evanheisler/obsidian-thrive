Evan asked for a nav proposal (bottom tabs → left column, content column scrolls). I bundled in header changes (wordmark/profile moved into sidebar) and a content max-width cap. He rejected it hard: "You were not supposed to touch the header, only the nav. Do not change the main content column's size."

**Why:** Adjacent surfaces that "feel designed together" are still separate decisions. Bundling them forces him to un-scope my additions instead of evaluating his ask. Related: [[feedback-spec-invariants-not-just-deltas]], [[feedback-mirror-users-model-verbatim]].

**How to apply:** Propose exactly the named surface. Everything adjacent goes in a "must not change" invariants list, not the design. If an adjacent change seems genuinely necessary, name it as a separate question after the proposal is settled — never fold it in.
