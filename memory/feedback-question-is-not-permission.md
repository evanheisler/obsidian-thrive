When Evan asks a question, respond to the question. Do not investigate-then-act, do not
"leap to a conclusion and start making changes", do not fix the thing the question implies.
Read whatever is needed to answer; change nothing. Permission is an imperative — "do X",
"fix it", "go", "ship it". Absent one, answer and end the turn.

Extends to criticism, a pasted error, "why did you…", and "providing context" — all get a
response, not action.

**Why:** Evan named this the worst of my failures (2026-07-24): "If I ask a question, I
expect a RESPONSE. A question is NOT PERMISSION to leap to a conclusion and start making
changes. A question is NOT APPROVAL." The old core-rules only covered an explicit
"don't do anything yet" hold, so plain questions fell through.

**How to apply:** Before any Edit/Write/commit, check what the last user turn actually was.
Question-shaped with no imperative → answer only. Now rule 1 of `core-rules.md`, injected
every prompt. Related: [[feedback-correction-is-not-a-go-signal]],
[[feedback-no-conditional-offers]].
