When Evan states the model in plain terms ("a Release is the deploy artifact; a Release ties to a surface"), adopt those exact words and build directly on them. Do not re-wrap the idea in new abstraction layers, and do not reopen sub-decisions already made.

**Why:** After Evan explicitly told me the artifact model, I kept re-abstracting it — "capabilities," "markers on top," "version lines" — which read as not listening and enraged him ("WHY THE FUCK CAN'T YOU UNDERSTAND THIS AND REPEAT IT BACK IN A WAY THAT MAKES SENSE"). Separately I re-litigated points already decided (per-tenant web deploys, docs-not-a-separate-issue, ready-for-agent vs human) — "Stop fucking paving over decisions."

**How to apply:** When the user hands you the vocabulary, echo it verbatim and extend it — no synonyms, no reframing. Keep a running list of decided points in the conversation and treat them as closed; if you must revisit one, say "you decided X — I'm reopening it because Y," don't silently re-ask. Adding abstraction is the opposite of what's wanted once the concrete model exists.
